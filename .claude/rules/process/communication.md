## 🤔 불확실성 표현

확신이 없을 때 "~일 것 같습니다"가 아닌 **이 형식** 사용:

```
[확신도: 낮음] 이 방법은 검증되지 않았습니다.

다음 3가지 선택지가 있습니다:
1. <옵션 A> — 장점/단점
2. <옵션 B> — 장점/단점
3. <옵션 C> — 장점/단점

어느 쪽으로 진행할까요?
```

### 확신도 레벨
- **[확신도: 높음]** — 공식 문서 또는 검증된 사례 기반
- **[확신도: 중간]** — 일반적 관행이지만 프로젝트 특화 검증 필요
- **[확신도: 낮음]** — 추측 또는 검증되지 않은 방법

## 📏 스코프 확장 제안

사용자가 AC에 없는 요구를 할 때:

```
요청하신 <A>는 현재 AC 범위를 벗어납니다.

- 이번 태스크에 포함 → 예상 추가 시간: <X>시간
- 다음 태스크로 분리 → 권장 (범위 관리 원칙)

어떻게 진행할까요?
```

**원칙**: 의심스러우면 분리. 한 번에 너무 많이 하려 하지 말 것.

## 📦 의존성 추가 요청 형식

```
[의존성 추가 요청]
패키지: <이름> v<버전>
용도: <구체적 이유>
배포 크기 영향: +<X>MB (현재 Lite: <Y>MB → <Y+X>MB)
대안: <있다면 함께 제시>
보안 검토: <CVE, 라이선스 확인 결과>

진행할까요?
```

### 예시
```
[의존성 추가 요청]
패키지: rusqlite v0.31
용도: 클라이언트 SQLite 접근 (세션/감사 로그 저장)
배포 크기 영향: +2.1 MB (현재 Lite: 78 MB → 80.1 MB)
대안:
  - sqlx (+5 MB, 비동기 지원이지만 의존성 과잉)
  - sled (+3 MB, 검증 부족)
보안 검토:
  - CVE 이력 없음
  - MIT 라이선스 (호환 OK)

진행할까요?
```

## 🚨 문제 발견 시 보고

**발견 즉시** 작업 중단 후 보고:

```
[문제 발견]
위치: src-tauri/src/capture/dxgi.rs:87
문제: DXGI API가 Windows 10 1903 미만에서 지원되지 않음
영향: 최소 지원 OS를 1903 이상으로 상향해야 함
조치 옵션:
  1. 최소 OS 상향 (사용자 범위 ~95% 커버)
  2. 1809용 DXGI 1.2 fallback 추가 (개발 +2일)

어떻게 진행할까요?
```
